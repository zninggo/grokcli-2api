系统设置

修改管理员密码、轮询策略与 sub2api / 维护参数

—

修改管理员密码

密码哈希写入数据库（hybrid 的 PostgreSQL）。环境变量 GROK2API_ADMIN_PASSWORD 仅用于首次无密码时自动导入，之后鉴权以数据库为准。

—

轮询与维护

账号选择策略与后台维护开关（立即生效）。间隔类改完保存后下一周期生效。

逗号分隔；后台每轮轮询其中一个，避免笛卡尔积打爆池子
默认 90；大池子可适当调高减轻刷新风暴
到期前多少秒开始刷新；默认 120
0 = 仅手动；≥60 为周期探测。默认 900
多轮对话绑定同一账号的缓存时长；默认 7200（2h）

账号出站代理池

账号池请求上游（聊天 / 测活 / Token 续期）走代理。按账号粘性选代理，与会话粘性配合更稳。协议注册可共用同一列表，也可单独配置。

—

未配置时直连上游。策略 round_robin 会对账号 ID 做稳定哈希（同账号固定出口）。

Relay / sub2api 参数

影响 Claude Code / sub2api 工具流、reasoning 兼容与长上下文压缩。单工具 + 间隔可避免 “Content block not found”。

sub2api / Claude Code 建议保持 1
仅 /v1/chat/completions；默认 0
推荐 0.08；纯 OpenAI 客户端可设 0
messages JSON 超过此长度时强制压缩；0=关闭（默认，优先智商）。仅当大会话导致上游失败时再设阈值
仅在开启压缩时生效：近乎完整保留最近 N 轮 tool；默认 32（智商优先）
仅开启压缩时：最近轮次单条 tool 上限（head+tail）；默认 48000

轮询 / 冷却 / 踢出策略

控制失败后冷却时长、临时 free-usage 软封禁、以及单次请求最多故障切换账号数。

free-usage 耗尽时仅封禁该模型，到期自动恢复
模型真正不可用时封禁时长，到期由探测自动复核
连续失败会按指数放大冷却（有上限与抖动）。临时额度错误优先模型软封禁；模型不可用会封禁该模型并冷却账号；自动模型探测会周期性复检，多次失败则踢出轮询。

sub2api 导入

—

配置 sub2api 管理端地址与登录后， 可在「账号」页将本地 Grok 账号批量导入到指定分组（平台 grok / OAuth）。 优先使用本地 access/refresh token 直推；无 token 时回退 SSO→OAuth。

协议注册导入本地后立刻推送到上方分组；需同时启用 sub2api 并填好 URL/登录
本地→sub2api 同时推送几个账号；默认 4
写入 sub2api 每个账号的并发容量；在线推送与导出数据均生效。默认 3
sub2api account.priority；默认 50
sub2api 计费/限速倍率；默认 1.0

  

    保存后到「账号」页：可「导入 sub2api（选中/全部）」在线推送；或「导出 sub2api 数据」下载官方 sub2api-data 备份，在 sub2api 管理台「导入数据」上传。
    勾选「注册成功后自动入库 sub2api」后，协议注册导入本地账号会立刻推送到上方配置的分组。
